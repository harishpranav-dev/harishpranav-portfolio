// how i build with ai

AI is my pair,
not my autopilot.

AI didn't replace the engineering — it compressed the boring parts so I can spend real time on what actually matters: the problem, the UX, and the edge cases nobody thinks about. Here's where it sits in my full-stack flow.

01

Ideation & specs

I pressure-test the problem out loud, poke holes in my own scope, and draft tight specs — so I'm building the right thing before I write a line.

02

Frontend

Scaffold components fast, then refactor by hand. AI's real value here is catching accessibility and responsive edge cases I'd otherwise miss at 2am.

03

Backend & APIs

Draft route handlers, gut-check error handling, and generate test payloads — then I own the logic that actually touches user data.

04

Database

Model schemas, sanity-check Prisma migrations, and rewrite the queries that are quietly doing too much before they hit prod.

05

Debugging

Paste the stack trace, get three hypotheses in seconds, then verify each one myself. Faster loops, same standard of proof.

06

Learning

New library, new pattern, new "why is this like this?" — AI is the fastest tutor I've had. Being self-taught, that's the multiplier.

!

I read every line before it ships. AI accelerates me — it doesn't get to be careless for me. The judgment, the taste, and the "does this actually help the user" call stay mine.
